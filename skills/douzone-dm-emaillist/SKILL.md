# 더존 DM 이메일 리스트 추출 스킬

## 개요

세 가지 소스 파일에서 이메일을 우선순위에 따라 채우고, 대량메일 발송용 Excel을 생성한다.

**주요 기능**
1. **DM 이메일 리스트**: upgrade 대상 고객사 기준, NSM → 재계약 순으로 이메일 보완
2. **고객 구성 비율 분석**: WEHAGO/ICUBE/Bizbox Alpha/Amaranth 10 제품별 비율
3. **솔루션 고객사 이메일 추출**: ICUBE·Bizbox·A10 보유 고객사 별도 추출

---

## 입력 파일 판별

| 파일 유형 | 식별 키워드 / 시트명 | 핵심 컬럼 |
|---|---|---|
| **Upgrade 리스트** | `위하고교체(신규)` 시트, `a10upgrade` | `사업자번호`, `거래처`, `이메일` |
| **NSM 고객현황** | `NSMCUS`, `NSM` | `사업자번호`, `담당자이메일`, `구매 제품군` |
| **재계약 리스트** | `recontract`, `wecrm`, `재계약` | `사업자등록번호`, `담당자 이메일` |

파일이 2개만 업로드된 경우 → 있는 소스만 사용하고 진행한다.

---

## 작업 모드 판별

업로드 파일 조합을 보고 자동으로 모드를 결정한다.

| 조합 | 실행 모드 |
|---|---|
| Upgrade + NSM (+ 재계약 선택) | **모드 A**: DM 이메일 리스트 생성 |
| NSM 단독 | **모드 B**: 고객 구성 비율 + 솔루션 이메일 추출 |
| 세 파일 모두 | **모드 A + B** 동시 실행 |

---

## 모드 A: DM 이메일 리스트 생성

`scripts/build_dm_list.py` 를 실행한다.

### 이메일 우선순위 (폭포수 방식)
```
1순위: Upgrade 파일 원본 이메일
2순위: NSM 담당자이메일  (사업자번호 매칭)
3순위: 재계약 담당자 이메일 (사업자번호 매칭)
미확보: 빈칸 유지, 출처='미확보'
```

### 출력 컬럼
| 열 | 컬럼명 | 설명 |
|---|---|---|
| A | 상호명 | 거래처명 |
| B | 사업자번호 | 원본 형식 유지 |
| C | 이메일 | 최종 확보 이메일 |
| D | 이메일 (복사용) | `email@domain.com,` 형식 — 셀 전체 복사해서 대량메일 수신자란에 붙여넣기용 |
| E | 이메일출처 | WEHAGO / NSM보완 / 재계약보완 / 미확보 |

### 99개 단위 색상 그룹
대량메일 발송 시 99개씩 끊어 보내는 작업을 지원하기 위해 행을 색상으로 구분한다.

| 그룹 | 색상 |
|---|---|
| 1 (1~99행) | 흰색 |
| 2 (100~198행) | 연노랑 `FFF9C4` |
| 3 (199~297행) | 연초록 `E8F5E9` |
| 4 (298~396행) | 연파랑 `E3F2FD` |
| 5 (397~행) | 연분홍 `FCE4EC` |

G열에 그룹 번호와 개수 레이블 표시.

## 모드 B: 고객 구성 비율 + 솔루션 이메일

`scripts/build_product_analysis.py` 를 실행한다.

### 제품 판별 기준 (`구매 제품군` 컬럼 텍스트 포함 여부)
- **WEHAGO**: `'WEHAGO'` 포함
- **ICUBE**: `'ICUBE'` 포함
- **Bizbox Alpha**: `'Bizbox'` 포함
- **Amaranth 10**: `'Amaranth 10'` 포함
- **기타**: Smart A, ERP-iU 등

### Sheet1: 고객구성비율
- 제품별 고객사 수, 비율(%)
- 파이차트 삽입
- 전체 합계 행 포함

### Sheet2: 솔루션고객사_이메일
솔루션(ICUBE/Bizbox/A10) 보유 고객사만 추출.

| 컬럼 | 내용 |
|---|---|
| 상호명 | 거래처명 |
| 사업자번호 | |
| 제품구성 | `ICUBE`, `Bizbox Alpha`, `Amaranth 10` 조합 |
| WEHAGO여부 | O / 빈칸 |
| 이메일 | 담당자이메일 |
| 이메일 (복사용) | `email,` 형식 |

---

## 사업자번호 정제 규칙

매칭 전 반드시 정규화:
```python
df['biz'] = df['사업자번호'].str.replace('-', '').str.strip()
```
재계약 파일은 `사업자등록번호` 컬럼명 사용 → 동일하게 정제.

## 출력 파일 명명 규칙

| 조건 | 파일명 |
|---|---|
| 담당자명이 파일명에 있는 경우 | `{담당자명}_dm_list_{날짜}.xlsx` |
| 담당자명 불명 | `dm_list_{날짜}.xlsx` |
| 모드 B | `고객구성비율_솔루션이메일_{날짜}.xlsx` |

날짜: 오늘 날짜 `YYYYMMDD`

## 결과 보고 형식

작업 완료 후 반드시 아래 표를 출력한다.

```
=== [담당자명] DM 리스트 결과 ===
전체 대상:     N개사
이메일 확보:   N개 (WEHAGO: N / NSM보완: N / 재계약보완: N)
이메일 미확보: N개
99개 그룹:    N그룹
```

미확보가 있을 경우: "미확보 N개는 수동 확인이 필요합니다." 안내.
